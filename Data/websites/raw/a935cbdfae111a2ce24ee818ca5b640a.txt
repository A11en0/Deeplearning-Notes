          <h2>Highlights in OpenWrt 19.07.0-rc2</h2>
<p>The OpenWrt community is proud to announce the second release candidate of the upcoming OpenWrt 19.07 stable version series. It incorporates <a href="https://openwrt.org/releases/19.07/changelog-19.07.0-rc2">126 commits</a> since the <a href="https://openwrt.org/releases/19.07/notes-19.07.0-rc1">previous release candidate 19.07.0-rc1</a>.</p>
<p>With this release, the OpenWrt project brings all supported targets back to a single common kernel version and further refines and broadens existing device support. It also provides initial support for the new  <code>ath79</code>  target, the future device tree based successor of the popular  <code>ar71xx</code>  target. For 19.07, both targets are still built, but it is recommended to switch to the  <code>ath79</code>  target whenever possible: future releases of OpenWrt will drop support for the  <code>ar71xx</code>  target.</p>
<p>Highlights of this release candidate since the previous OpenWrt <a href="https://openwrt.org/releases/19.07/notes-19.07.0-rc1">19.07.0-rc1 release candidate</a> are:</p>
<ul>
<li>
<p><strong>Linux kernel</strong>  updated to versions  <strong>4.14.156</strong>  (from 4.14.151 in v19.07.0-rc1)</p>
</li>
<li>
<p><strong>GCC update</strong>  to version  <strong>7.5.0</strong>  (from 7.4.0 in v19.07.0-rc1)</p>
</li>
<li>
<p><strong>Hostapd update</strong>  to version  <strong>2.9</strong></p>
</li>
<li>
<p><strong>Device support bugfixes</strong>  for many devices</p>
</li>
<li>
<p><strong>Security fixes</strong>  for curl, e2fsprogs, expat, wolfssl, intel-microcode</p>
</li>
<li>
<p><strong>Sysupgrade bugfix</strong>  for an issue that could cause sysupgrade to silently fail to upgrade</p>
</li>
<li>
<p><strong>Added support</strong>  for the following devices: YunCore XD4200 and A782, TP-Link Archer C60 v1/v2, ALFA Network Quad-E4G and R36M-E4G, AVM FRITZ!Repeater 1200</p>
</li>
<li>
<p><strong>Disable image generation</strong>  for several devices due to insufficient flash space for the default set of packages. Images can still be built with the help of the <a href="https://openwrt.org/docs/guide-user/additional-software/imagebuilder">ImageBuilder</a> or <a href="https://openwrt.org/docs/guide-developer/using_the_sdk">SDK</a> with a more minimal set of packages</p>
</li>
</ul>
<p>When upgrading from 19.07.0-rc1, if you are affected by the sysupgrade bug, you need to manually edit  <code>/lib/upgrade/stage2</code>  and add  <code>/usr/share/libubox/jshn.sh</code>  to list of files in the  <code>install_file</code>  line (around line 51).</p>
<p>Known issues:</p>
<ul>
<li>
<p>Sysupgrade from  <code>ar71xx</code>  to  <code>ath79</code>  and vice versa is not officially supported, a full manual reinstall is recommended to switch targets for devices supported by both  <code>ar71xx</code>  and  <code>ath79</code></p>
</li>
<li>
<p>Images for some device became too big to support a persistent overlay, causing such models to lose configuration after a reboot. If you experience this problem, please report the affected device and consider downgrading to OpenWrt 18.06 or using the Image Builder to pack a smaller custom image</p>
</li>
<li>
<p>Some optional GUI packages crash with an error about missing “cbi.lua”, install the  <code>luci-compat</code>  package to fix these</p>
</li>
<li>
<p>Possible Wi-Fi issues with ath10k-based boards. If you encounter such an issue, please <a href="https://openwrt.org/bugs">file a bug report</a> against openwrt-19.07.</p>
</li>
</ul>
<p><a href="https://openwrt.org/releases/19.07/notes-19.07.0-rc2" class="onebox" target="_blank">https://openwrt.org/releases/19.07/notes-19.07.0-rc2</a><br>
